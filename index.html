HTML and CSS

1. HTML is the structure/p>

2. CSS is the style

3. Javascript,not to be confused with Java, changes elements and add interactivity within simple HTML.

HTMLS Structure

The strutcute of HTML coding is similar to a tree, with one limb branching off to another. The parent of those limbs being those aspects which came before it. This tree like structure is also called "document object mode" or DOM.

HTML, is box oriented. What is meant by this is that, HTML reads and sets up in box formats, regardless of the actual shape of an object. For example; if a circle is the original shape, HTML will read is as a box around the circle. A circle can be created using element tools to resize and shape.

CSS can be used with HTML to efficiently add style to HTML elements. It is important to note, that files for a specific project should be saved in the same folder - include CSS, HTML and image files together- otherwise CSS will not appear to have made any style changes tot he afore mentioned.

CSS Information

1. CSS is the language used to give an HTML document style. CSS can manipulate font, margins, background color, and image positioning to name a few.

2. CSS uses specific syntax and rules seperate from HTML.

3. HTML styling can be manipulated within in HTML, however this can be cumbersome and is not efficient in most cases. CSS makes the case for efficiency of style manipulation.

Avoiding repetition is important because it allows for efficient use of time both in creating coding and in changing coding at a later date. One way to create efficient code is to group together elements that will have the same look and style. For example on this particular page, most of the style applied, used classes that applied to each segement, and each segment has similar characterisitcs; i.e. background color. To achieve this I wrote out general classes like outline, title-two (for secondary titles) etc, that applied to each segement. The end result is that I only needed to write out code once for all segements. Should any changes need to occur within the segements at a later date, identifying changes will be easy for both the original programmer and any new person wishing to change styling.
